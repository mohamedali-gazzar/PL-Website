THE REFERENCE RECIPE (reproduce this quality & approach)
================================================================================

1. EXACT TECH STACK — use these unless I explicitly ask otherwise
--------------------------------------------------------------------------------
- Framework: Next.js 14.2, App Router, React 18.3.1, JavaScript (JSX, NOT
  TypeScript), npm.
- Styling: styled-jsx (built into Next) — a scoped `<style jsx>{`...`}</style>`
  block inside each component — PLUS a single global app/globals.css holding
  design tokens (CSS variables), reset/base type, and shared utility classes.
  NO Tailwind, no CSS-in-JS library, no component/UI kit.
    * SSR styled-jsx through a StyledJsxRegistry client component
      (useServerInsertedHTML + createStyleRegistry from "styled-jsx") wrapped
      around {children} in app/layout.jsx -> NO flash of unstyled content.
    * Inline a tiny critical-CSS string in <head> (html/body background + base
      font/color) so first paint is already dark and branded.
    * styled-jsx gotchas: (a) NEVER put an SVG data: URI inside a <style jsx>
      block — it silently drops the whole stylesheet; put data-URI backgrounds
      in globals.css or an inline style. (b) styled-jsx scope classes are NOT
      applied to <Link>/component children — for rules that must hit them, use
      :global(...).
- Animation: GSAP 3.15.0 (+ the ScrollTrigger plugin) and Lenis 1.3.25 (smooth
  scroll). No Framer Motion.
- Icons: a small custom inline-SVG `Icon` component (Lucide-style geometry,
  stroke="currentColor", one consistent stroke width). NO icon package. NEVER
  emoji as icons.
- Fonts: next/font/google — a bold geometric display face for headings
  (reference uses Montserrat 400-900, standing in for a licensed face) + a clean
  humanist sans for body (Poppins 300-700), exposed as --font-head / --font-body.
  Optionally an @font-face for a licensed brand font that auto-overrides when the
  files exist in /public/fonts.
- Path alias: @/* -> ./* (jsconfig.json).
- Config patterns: next.config with reactStrictMode:false, a distDir overridable
  via NEXT_DIST_DIR env (so a preview build never clobbers .next),
  images.remotePatterns for the live domain, and redirects() for renamed slugs.
  ESLint via eslint-config-next.


2. STRUCTURE & LAYOUT SYSTEM
--------------------------------------------------------------------------------
- app/layout.jsx — fonts, <html>/<body>, metadata (title/description/OpenGraph/
  Twitter/icons, metadataBase = the live https://www... URL), viewport
  (themeColor), inline critical CSS, StyledJsxRegistry, and a small ChunkReload
  guard component.
- app/registry.jsx — the styled-jsx SSR registry.
- app/globals.css — the :root design-token block, reset, base typography, shared
  classes (.container, .eyebrow, .section-title, .sec-head, .btn / .btn-primary /
  .btn-ghost, .reveal, .field / .err / .form-err), and the fluid large-screen
  scaling rule.
- app/page.jsx — the home page: composes the section components, wraps everything
  in the smooth-scroll provider, plays a one-time intro preloader, and
  CODE-SPLITS: above-the-fold + any pinned/scrubbed sections are imported eagerly
  (their scroll math must be stable at first paint); trailing sections use
  next/dynamic with a height-placeholder `loading` fallback to avoid layout shift.
- One folder per route under app/ (about, contact, careers, locations, collection/
  category pages, a dynamic products/[slug], etc.).
- lib/content.js — THE SINGLE SOURCE OF TRUTH. Every string, stat, nav item,
  product, project, value, location, social link, etc. is an exported constant.
  Pages/components import from here and stay presentational; nothing user-facing
  is hard-coded in JSX. This is what makes the whole site re-skinnable per company.
- components/ — shared UI, with components/sections/ for the home sections.
- public/img (optimized .webp), public/video (hero loop).


3. REUSABLE COMPONENTS & PRIMITIVES (recreate equivalents)
--------------------------------------------------------------------------------
- SmoothScroll — boots Lenis (duration ~1.1, exponential ease, smoothWheel),
  pipes lenis.on('scroll', ScrollTrigger.update) and runs lenis.raf off
  gsap.ticker so pinned/scrubbed sections stay in sync; sets
  history.scrollRestoration='manual' + scrollTo(0,0); exposes window.__lenis; and
  is FULLY DISABLED under prefers-reduced-motion.
- PageShell — interior-page wrapper = SmoothScroll + ambient background + Nav +
  <main> + Footer.
- PageHero — interior page header: optional background image + dark veil, small
  eyebrow, big uppercase title with one accent-colored word, lead paragraph.
- Primitives: Reveal (IntersectionObserver adds .in; CSS does the fade/rise:
  .reveal{opacity:0;transform:translateY(28px);transition:...} /
  .reveal.in{opacity:1;transform:none}, with a per-item delay for stagger),
  CountUp (animates a number when it scrolls into view, reduced-motion-aware),
  Field (labelled input + inline error + optional leading Icon).
- Nav — fixed header, transparent -> blurred/bordered on scroll (is-scrolled),
  logo left, center links with hover dropdowns, right-side CTA buttons (as <Link>
  styled via :global), mobile burger -> slide-in drawer with staggered items. Nav
  tree lives in lib/content.js.
- Footer — dark multi-column footer (brand blurb + socials, navigate, locations,
  contact/CTA), content-driven.
- Icon — the inline-SVG set. MagneticButton — pointer-follow magnetic hover.
  Preloader — one-time cinematic intro (GSAP timeline), gated by a module-scoped
  flag so it plays once per full page load (not on in-app navigation) and
  stops/starts Lenis.
- Signature ambient motif — a subtle canvas/particle "living background" behind
  the page + a thin animated side rail/progress accent. For a new industry, swap
  the metaphor (electricity -> whatever fits) but keep the idea: a quiet living
  background plus one signature animated line/element.
- Domain components — collection/grid view, card grid, image gallery with
  thumbnails, an animated values diagram, a history timeline, a before/after
  comparer, a partner/client logo marquee. Rebuild the equivalents the new
  sections need.


4. DESIGN TOKENS (globals.css :root) — keep the SYSTEM, swap the VALUES
--------------------------------------------------------------------------------
- Surfaces: --bg (near-black base, ref #050506), --bg-2, --bg-3 (progressively
  lighter dark card surfaces), --line (rgba(255,255,255,.08) hairline borders).
- Text: --text (near-white), --text-dim, --text-faint.
- ONE strong accent + bright/deep variants + a --glow shadow (ref orange
  #e8722a). Replace with the new brand's accent; keep the discipline of one
  confident accent on a dark canvas.
- Type: --font-head, --font-body. Layout: --container: 120rem (=1920px at base,
  centered), --pad: clamp(1.1rem,2.5vw,2.25rem), --ease: cubic-bezier(.22,1,.36,1),
  a z-index scale.


5. DESIGN LANGUAGE TO PRESERVE
--------------------------------------------------------------------------------
- Cinematic dark, premium, immersive, slightly industrial — one confident accent,
  lots of near-black negative space. Not flat, not corporate-generic.
- Layout: full-bleed hero; wide, tall sections with generous clamp() vertical
  padding; alternating left/right compositions; content in a centered max-width
  container that still reads full-bleed on laptops.
- Typography: big UPPERCASE display headings (700-900), tight leading; calm
  sentence-case body in a lighter sans; small letter-spaced accent eyebrows
  labelling each section.
- Cards: dark --bg-2/-3 panels, 1px --line border, ~14-20px radius, some with a
  soft radial accent-tinted glow; hover = lift (translateY(-4/-5px)) + accent
  border.
- CTAs: pill buttons (border-radius:100px); primary = solid accent + --glow;
  secondary = ghost/outline; hover translateY(-3px) + stronger glow. One primary
  CTA per view.
- Nav/Footer: fixed translucent nav that blurs on scroll; dark multi-column footer.
- Imagery: photos at reduced opacity under dark gradient veils so text stays
  legible; a faint grid/scanline overlay for texture.
- Motion personality: smooth Lenis inertia scroll; reveal-on-scroll fades/rises
  (stagger ~0.06-0.1s); scroll-scrubbed parallax and a few PINNED sections; number
  count-ups; SVG stroke-dashoffset draw-on diagrams; magnetic buttons; a signature
  animated "energized line". UI transitions 150-400ms, longer for hero/scroll. ALL
  of it must degrade gracefully under prefers-reduced-motion.
- Responsive: mobile-first; systematic breakpoints (~640 / 760 / 860 / 1024 /
  1280); height-based media queries where a hero must fit the first screen.
- LARGE-SCREEN BEHAVIOR (critical): the root font-size scales up above 1920px —
  @media (min-width:1921px){ html{ font-size: clamp(16px, min(calc(16px +
  (100vw - 1920px)*0.006), 2.1vh), 24px) } } — so everything sized in rem/clamp
  grows together and the page FILLS big monitors and deep browser zoom-out instead
  of shrinking into a small centered block with huge empty margins. Size EVERYTHING
  in rem/clamp, never fixed px (fixed px "freezes" and looks tiny on large screens).
  No zoom, no transform:scale.


6. STORYTELLING / SECTION RHYTHM — adapt the CONTENT, keep the ARC
--------------------------------------------------------------------------------
Reference home flow:
  (0) one-time cinematic preloader ->
  (1) full-bleed HERO (hero visual/video, one bold uppercase promise, primary +
      ghost CTAs, a 4-up STATS BAR pinned to the bottom of the first screen) ->
  (2) flagship work / projects ->
  (3) core offerings / product lines ->
  (4) history / milestones timeline ->
  (5) values (animated diagram) ->
  (6) a "what we stand for" statement + parallax image ->
  (7) a signature proof/demo (e.g. before/after) ->
  (8) partner/client logo wall ->
  (9) conversion CTA band ->
  (10) a human/legacy closing note ->
  footer.
Adapt each beat to the new field: hero = the company's boldest promise;
"projects" = their proof-of-work (case studies, portfolio, menu, fleet, listings,
results...); "product lines" = their core services/products. KEEP the credibility
beats (stats, timeline, certifications, partners, testimonials) and a strong
closing CTA. Principle: open with impact -> alternate visual rhythm -> build
credibility progressively -> keep the eye moving with motion -> close on conversion.
